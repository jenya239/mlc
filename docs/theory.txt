Отличный вопрос! К сожалению, эти тонкости разбросаны по разным источникам. Вот самые ценные:

## Книги с архитектурными деталями

### 1. **"Engineering a Compiler"** - Cooper & Torczon
```
Глава 5: Intermediate Representations - детально про уровни IR
Глава 8: Introduction to Optimization - про организацию passes
Глава 15: Building a Runnable Program - про pipeline
```

### 2. **"Modern Compiler Implementation in ML/Java/C"** - Appel
```
Особенно ценно:
- Tree IR vs Graph IR (глава 8)
- Dataflow analysis framework (глава 10)
- SSA form (глава 19)
- Functional IR (глава 15)
```

### 3. **"SSA-based Compiler Design"** - Rastello & Bouchez (2022)
```
Современная книга именно про архитектуру:
- Глава 3: SSA Construction 
- Глава 11: Instruction Selection как rules
- Глава 18: Compiler Infrastructure (LLVM, GCC)
```

## Академические статьи (must read)

### Про архитектуру компиляторов:
1. **"A Nanopass Framework for Compiler Education"** (2006)
   - Dipanwita Sarkar, Oscar Waddell, R. Kent Dybvig
   - Описывает подход с микро-проходами

2. **"The Glasgow Haskell Compiler"** (2012) 
   - Simon Marlow, Simon Peyton Jones
   - Детально про pipeline GHC

3. **"Bringing the Web up to Speed with WebAssembly"** (2017)
   - Про дизайн WASM и его IR

### Про Visitors и трансформации:
4. **"Visitor Combination and Traversal Control"** (2001)
   - Joost Visser
   - Продвинутые паттерны visitor

5. **"Attribute Grammars and Visitors"** (1998)
   - Как правильно организовать visitors

## Мой совет по порядку изучения:

1. **Начните с Nanopass paper** - даёт концептуальную основу
2. **Прочитайте исходники Opal или Crystal** - простые, на Ruby
3. **Изучите V8 blog posts** - практические решения
4. **Посмотрите LLVM talks** - продвинутые техники
5. **Почитайте Rust Compiler Guide** - современный подход

Самое ценное - это комбинация теории (papers) и практики (исходники). К сожалению, единой книги "Архитектурные паттерны компиляторов" пока нет.
