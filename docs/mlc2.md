## A4. Параметры по умолчанию

### Соответствие `MLC.md` (§A4)

| Требование | Состояние |
|------------|------------|
| Синтаксис `name: T = expr`, хвостовые defaults | Сделано: Ruby + mlcc (парсер; порядок — диагностика) |
| C++: нативные `default` только в **первом** объявлении (у нас: proto/forward, не в def) | Сделано |
| Ограничение: «compile-time или вызов без побочных эффектов» | **Частично:** везде заведён **MVP** — разрешены **литералы** (и в Ruby: `true`/`false` как ветка `VarRef`, где задано). Произвольные const-выражения, `const fn`, вызовы `constexpr` — **вне** текущей реализации. |
| `extern fn` без defaults | Запрещено на обоих уровнях |
| Generics + defaults | Пока **запрещено** (явное сообщение; отдельный подсчёт подстановки типов) |

### Паритет Ruby ↔ mlcc (важно)

- **Верхнеуровневые** `fn`: checker mlcc и Ruby-пайплайн согласованы (арность, defaults, тип default).
- **Методы в `extend { ... }` и объявления в `type T { ... }`:** в Ruby сигнатуры/defaults проходят через `function_reducer` на синтетических `FuncDecl`. В **mlcc** вход `check` в `check.mlc` **не** обходит вложенные `DeclFn` внутри `DeclExtend` / `DeclTrait` (ветки пустые). Следствие: правила A4 и прочие проверки **тел функций** для таких объявлений в mlcc **не** повторяют объём Ruby, пока checker не научится рекурсивно обходить вложенные декларации. Codegen self-hosted по-прежнему может затрагивать эти функции; источник истины для **семантики** в таком месте сейчас — Ruby-конвейер.
- **Дефолты в сигнатурах трейта как C++ concept:** в §A4 плана (отдельно) отмечалось, что «чистая» сигнатура трейта с default может усложнить генерацию; отдельное решение (запрет/разрешение + правила) не зафиксировано в `MLC.md` — при появлении в парсере нужен явный дизайн.

### Критерий «фича закрыта» по `MLC.md` §Процесс

- `rake test_mlc` и `rake test_compiler_mlc` (сборка `compiler/tests/`) — целевой инвариант. Сбой окружения (например, отсутствие `minitest` в тестовом раннере) не отменяет требований; прогон принимается на машине, где тесты установлены.

### Следующие этапы (не блокируют закрытие A4-MVP)

1. Расширить класс допустимых default-выражений до const-подмножества, согласованного с C++.
2. Generics + defaults: вывод и проверка при подстановке `T`.
3. Рекурсивный **check** для вложенных `fn` в mlcc (`DeclExtend` / `DeclTrait` / при необходимости импорты) — выровнять паритет с Ruby.
4. При необходимости: default в сигнатурах трейта + стратегия concept/codegen.
