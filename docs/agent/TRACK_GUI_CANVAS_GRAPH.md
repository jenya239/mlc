## Status: open, design-only, низкий приоритет (условно — «если сделаем»)

## Зачем отдельный трек

Текущий GUI v0 ([../GUI.md](../GUI.md)) — честный immediate-mode на
screen-space координатах (кнопка, layout-курсор). Требование пользователя
— **другая архитектура**, не расширение v0: масштабируемое поле (десятки/
сотни/тысячи узлов), pan/zoom, связи между произвольными точками. Если не
зафиксировать сейчас — риск дальше наращивать v0/v1 виджеты на
несовместимых допущениях (screen-space `Rect`, no culling, no instancing)
и потом переписывать.

## Требования (по значимости)

| # | Требование | Сегодня | Нужно |
|---|-----------|---------|-------|
| 1 | Camera/viewport affine-трансформация (pan+zoom) | Нет — `layout.mlc` `Rect`/`Point` только screen-space `i32` | Логические координаты сцены + одна affine-трансформация при рендере/hit-test |
| 2 | Spatial index для culling (quadtree/grid) | Нет — draw all every frame | Обязательно для "ultra-fast" на сотнях+ узлов — иначе деградация с ростом сцены |
| 3 | Retained-граф с dirty-tracking | Нет — чистый IM (пересчёт layout каждый кадр) | Перекладывать/перетесселлировать только изменившееся; IM ок для одной кнопки, не для большой сцены |
| 4 | Batched/instanced draw (rect+текст на узел) | Нет — `gl_renderer.mlc` per-shape triangle helper | GPU instancing для тысяч узлов за кадр |
| 5 | Рендер связей (bezier между произвольными точками) | Нет примитива | Batched tessellation, отдельно от rect/text примитивов |
| 6 | Текст при произвольном zoom | **Готово** — MSDF `RenderMode=MsdfRgb8` (`pixel_size≥28`), single atlas entry per glyph независимо от масштаба | — |
| 7 | Generic граф-модель (узлы/порты/связи произвольных сущностей) | Нет | MLC record/sum types, отделённые от рендерера (рендерер знает только геометрию+текст) |

## Decision — отложено

Не фиксировать API/структуры до решения строить: слишком рано без
конкретного GUI_FRAMEWORK v1 контекста (виджеты/дерево ещё не спроектированы
за пределами v0 кнопки). Когда трек активируется — STEP=1 Decision должен
закрыть: формат сцены (retained tree vs ECS-подобный), формат spatial index
(quadtree vs uniform grid — зависит от распределения размеров узлов),
координатная точность на больших логических полях (camera-relative
rendering — вычитать позицию камеры перед отправкой в GPU, не копить
float-ошибку).

## Out of scope (пока)

- Реализация — этот трек не авторизует код, только фиксирует требование.
- Совместимость с текущим `misc/gui/` v0 API — вероятно новый параллельный
  слой (`misc/gui/canvas/`?), не расширение layout.mlc/button.mlc заплатками.
